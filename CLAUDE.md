# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

grackle is a local-first live code visualizer for Python: static graph from `ast`, runtime overlay via `sys.monitoring` (Python 3.12+) over a `127.0.0.1` WebSocket, React + Sigma.js frontend.

Status: **active solo development, contributions closed** (see `CONTRIBUTING.md`). Treat external PRs and issues as out of scope.

## Repo shape

pnpm workspace monorepo with three packages plus shared schema:

- `packages/agent/` — Python WebSocket server + adapters (`uv`-managed, hatchling build, `grackle` CLI entry point)
- `packages/frontend/` — React 19 + Vite + Sigma.js + Zustand
- `packages/shared-types/` — JSON Schema is the **single source of truth**; codegen emits TS interfaces *and* Python TypedDicts

JSON Schema → TS + Python codegen is the seam that prevents protocol drift. Generated files (`packages/shared-types/src/generated/`, `packages/agent/src/grackle/_generated/`) are **gitignored** — run `pnpm codegen` after a fresh clone or schema change. The hand-written `packages/shared-types/src/messages.ts` is the canonical public API; the generated TS is a sanity-check artifact and is not re-exported.

Run from the repo root unless noted.

# Bootstrap
pnpm install

pnpm codegen                              # required after fresh clone or schema change

## Non-obvious conventions

- **POSIX path discipline.** All path-bearing fields that cross the wire or persist (node IDs, annotation keys, manifest entries) must be POSIX-relative strings — `services/auth.py`, never `services\auth.py`. Use `grackle.paths.to_posix(p, root)`; do not call `.relative_to()` directly outside `paths.py`. A single missed call site silently diverges IDs between macOS and Windows. The `ruff PTH` ruleset and a path-discipline lint test guard this.
- **Open strings, not enums, on extension surfaces.** `language`, node `kind`, edge `kind`, trace `type` — all open `str`. Unknown values are ignored, not errors. See ADR-0004. The `KNOWN_*` `as const` arrays / `_canonical_*` validators in `kinds.py` are display-time conveniences, not gatekeepers.
- **Generated files have `_generated/` paths and are excluded from lint/typecheck** (`ruff exclude`, `mypy exclude` in `pyproject.toml`). Never edit them by hand; edit the schema and re-run `pnpm codegen`.
- **Lefthook enforces things locally before push.** `pre-commit` runs Biome + Ruff + schema parity; `pre-push` runs full typecheck + frontend tests + `mypy --strict` + pytest. If a hook fails, fix the underlying issue — don't `--no-verify`.
- **Atomic writes** — write to `.tmp`, then `Path.replace()` (not `Path.rename()` — Windows `rename` fails on existing target; this caused commit `3c31dca`).
- **Bind only to `127.0.0.1`.** Never `0.0.0.0`. Local-first is a product invariant, not a default.
- **Versions are single-source.** `__version__` is read from `importlib.metadata`; don't add string literals that duplicate `pyproject.toml`'s `version`.

**Phase 10.1 shipped at main=`9189497` (PR #49).** `packages/agent/src/grackle/python_runtime/value_repr.py` — a pure, security-hardened value→string formatter with no tracer/wire/CLI dependency, shipped standalone so the phase's most security-sensitive code is reviewable in isolation before 10.2 wires it into the tracer's hot path. Subclasses `reprlib.Repr` with an **exact-`type(x)` dispatch table** built once at import time (module-level `_DISPATCH`, not rebuilt per call) instead of stock reprlib's name-based dispatch — kills the "class named `list`" spoofing hole in one move, since a subclass or lookalike never matches an exact-type key. **Never invokes an arbitrary user `__repr__`** (the only in-process defense against a slow/hanging/side-effecting one) and **never consumes a lazy generator/iterator**. Fixes five stock-`reprlib` traps (name-spoofing dispatch, `sorted()`-on-keys invoking user `__lt__`, unbounded `repr_instance`, the Python 3.11+ huge-int `ValueError`, unbounded bytes materialization). Dataclass fields are redacted **before** the value is read at all (not just before it's shown), and the `__slots__` fallback accepts only a genuine `types.MemberDescriptorType` — never a `property`/`cached_property`, which would run arbitrary user code. A total character budget (`repr1`, leaf-charged only — container wrapper punctuation is unbudgeted since it's already bounded by the pre-existing item-count/depth caps) bounds nested-assembly work to a small constant multiple of `max_len` regardless of nesting depth. Name-based redaction (`password`, `token`, `api_key`, `authorization`, etc., `-`/`_` separator-normalized so header-style names like `X-Api-Key` redact) runs before any repr machinery touches a value. An xhigh multi-agent `/code-review` pass (10 finder angles → adversarial verify → sweep) found 11 real findings pre-merge; a follow-up independent verification pass (11 more agents, one per finding) then caught 2 residual bugs introduced by the first round of fixes (character-budget double-counting; a `slice`-with-non-int-component crash risk) plus 3 test-coverage gaps, all closed before merge. 66 tests, each verified to fail against the pre-fix code. CI green Ubuntu + Windows.

**Phase 10.2 shipped at main=`e304fa7` (PR #51).** Wires 10.1's `value_repr.py` into the Python `sys.monitoring` tracer (`python_runtime/tracer.py`) and lands the first wire-schema change since Phase 8.3: a first-class typed `values` field on `TraceEvent` (`{args?: ArgValue[], ret?: string, ret_truncated?: boolean}`), hand-synced across `trace.schema.json`, `messages.ts`, and `adapters/base.py`'s TypedDicts (ADR-0025 documents this three-way sync as review-guarded, not parity-guarded — `check-parity` only diffs generated-artifact bytes and message-`type` consts, never these hand-written shapes). **Mechanism:** returns are free (`retval` handed straight to `PY_RETURN`); args use the **verified-frame technique** — `sys._getframe(1)` called directly inside `_on_call` (never in a nested helper), `frame.f_code is code` checked before trusting `f_locals`, degrading to no-args capture on any mismatch so the call/return event is *always* emitted. Only declared parameters are read (`co_argcount`/`co_kwonlyargcount`/`CO_VARARGS`/`CO_VARKEYWORDS`), with dot-prefixed synthetic names (a generator expression's implicit `.0` iterator arg) filtered out. **Consent/sampling:** opt-in and default-off (`--capture-values`), Python-only (clean `UsageError` otherwise), a per-`node_id` capture budget (`--capture-first-n`, default 100, shared between a node's call and return events) that bounds *capture* only — never event *emission* — plus size caps (`--max-value-len`/`--max-value-items`/`--max-value-depth`) and name-based redaction (`--no-redact` escape hatch, reusing 10.1's `is_sensitive_name`). A default (non-capturing) run stays byte-identical: `new_trace_event`'s trailing `values` param is omitted from the dict, not set to `null`, when absent. ADR-0025 explicitly records the data-at-rest privacy surface (captured values, even redacted, persist to `-o` JSONL / `--stream` recordings / the session store). **Phase 10.4 shipped at main=`96c172c` (PR #55).** The first "why files connect" chunk (ADR-0026), fused with 10.1–10.3's "why they fire" half. Every static edge across all four parsers (`{python,typescript,go,rust}_parser/visitors.py`) and cross-language edges (`cross_language.py`) now carries a 1-based `metadata.line` — the justifying import/call/inherit/implements/route/spawn source line — in the edge's already-open `metadata` bag, so **no wire-schema change** (`check-parity` stays a no-op, per ADR-0004's open-metadata posture; a deliberate contrast with 10.2's typed `values` field). **Mechanism:** Python `ast` line numbers are already 1-based; tree-sitter rows are `node.start_point[0] + 1`. Call edges in TS/Go/Rust route through a `_CallCollector` that previously discarded position — now carries `(name, line)` tuples; `_emit_inherit`/`_emit_implements` gained a threaded `line` parameter from their positioned caller node. **The load-bearing trap:** all four `resolve_graph()` resolvers rebuild inherit/call edges on resolution, and the pre-existing resolved branch *replaced* the edge metadata (`{}` in TS/Go/Rust, `resolution.metadata` in Python) — silently dropping `line` for every successfully-resolved edge. Fixed by carrying the original evidence forward while dropping only the `resolved` marker (`{k: v for k, v in meta.items() if k != "resolved"}`), keeping the resolver idempotent. **Cross-language line threading turned out to be cheap, not risky:** an early exploration pass wrongly assumed hint positions came from the runtime adapters; they're actually static regex-over-source (`{python,typescript,go,rust}_parser/hints.py`, `extract_hints`), so the line is just the match offset (`source.count("\n", 0, m.start()) + 1`) — uniform across all four languages, zero tracer surface touched. **Frontend:** a new **`EdgeEvidencePanel`** (right-sidebar order 25) triggered by a new Sigma `clickEdge` handler or by selecting a node (lists its in/out edges); the snippet is derived client-side from source already fetched via `useSource` (the wire carries only the line, not text, keeping the parser change mechanical and the graph JSON lean). New store state `selectedEdge`/`sourceViewerTarget` + `selectEdge`/`jumpToSourceLine` actions; `SourceViewer` prefers the explicit target over the node-derived path/line (needed because a call edge's evidence line is deep in a function body, and an incoming edge's evidence lives in a *different* file). `MultiDirectedGraph` preserves parallel edges end-to-end, so two calls to the same target on different lines surface as two distinct, independently-jumpable rows. Line-less edges (Go method-set-synthesis `implements`, cross-language on a stale parse cache) degrade cleanly — no snippet, no crash; the stale-cache case is a known limitation documented in the ADR (clear `.grackle/cache/` to backfill) rather than a cache-format-version bump, to keep the chunk's scope tight. **Review:** manually verified end-to-end against `fixtures/tiny-polyglot` (14/14 edges carried correct lines, including both cross-language kinds; precise jump to a call site — not the def line — confirmed live), then an xhigh multi-agent `/code-review` (10 finder angles → 1-vote verify → sweep, ~29 agents across two runs after a session-limit interruption) found **one real correctness bug** — `selectEdge` cleared `selectedNodeId` but not `sourceViewerTarget`, so clicking a line-bearing edge then a line-less one left the SourceViewer pinned to the *previous* edge's file/line while the panel showed the new edge; independently flagged by 5 of the review's finder angles — fixed (`selectEdge` now clears the target too) and mutation-verified, plus two low-severity cleanups applied (a duplicated cross-language metadata-line block factored into `_edge_metadata()`; a no-op `{...MUTED}` style spread removed). A first CI run failed on Ubuntu only (jsdom lacks `Element.prototype.scrollIntoView`, newly exercised by an edge-evidence test whose target line resolves to a real element) — fixed with a one-line stub in the shared Vitest setup. CI green Ubuntu + Windows × Python 3.12 + 3.13 (4 legs).

**Phase 10.5 shipped at main=`0830b2f` (PR #57).** The payoff chunk fusing both Phase 10 halves: a selection-driven **`CausalPathPanel`** (right-sidebar order 27, adjacent to `EdgeEvidencePanel`) that answers "why did this node fire?" — pick a node, pick which firing when it fired more than once, and read the ancestor chain **root → … → THIS** with the argument values that drove each hop. **Frontend-only, no wire-schema change** — amends ADR-0026 §8 rather than opening a new ADR, per that ADR's own pre-commitment. **Mechanism:** a new pure `graph/causalPath.ts` — `firingsOf(events, nodeId)` enumerates a node's `call` events (capped at `MAX_FIRINGS=200`, scan breaks early rather than a full-prefix scan); `nearestFiring(firings, playhead)` picks a sensible initial default; `causalPathAt(events, callIndex, threadId)` is a thin, drift-guarded wrapper over 10.3's `ancestorStackAt` — the causal path for a firing **is** the open ancestor stack at the instant its `call` event opens, so no new reconstruction algorithm was needed. **The key new claim: truncation is a completeness banner, not a hard gate** (the one place 10.5 deliberately diverges from 10.3) — every rendered path is provably correct even from a >50k-truncated prefix, because `causalPathAt` only ever replays `[0, callIndex]`, which truncation past the cliff can't touch; only firing *enumeration* is bounded. **Per-hop navigation fuses all of Phase 10** via three independent single-action buttons (never combined, so one can't silently clobber another): `→ time-travel` (`setPlayhead` only), `select` (graph-guarded `selectNode`), `↳ call site` (`jumpToSourceLine` via the parent→hop edge's 10.4 `metadata.line`, looked up through an O(1) nested-Map index rather than a per-hop `graph.edges` scan). A new shared `graph/useSeekablePrefixState.ts` extracts `ValueInspectorPanel`'s seekable-load state machine (loading/error/unloaded/ready) so both panels stay in lockstep rather than risking a silent divergence. **Review (two full xhigh multi-agent rounds, 69 agents total) — the second round is the standout lesson:** round 1 (pre-commit, 29 agents) found and fixed six real issues, including a `callEdgeLineIndex` lookup that matched *any* edge kind sharing a (source, target) pair (an import/inherit edge could masquerade as a "call site") and a `nearestFiring` default that could go silently stale once the 200-firing cap was hit. Round 1's fix for a captureSeen re-scan was a `useRef` **latch** ("once true, stay true") — a fresh round-2 review on the *committed* PR (40 agents) found this latch **introduced two real regressions**: `useFullTrace` clears the prior session's prefix only in a post-commit effect (so a seekable→seekable session switch re-latched `true` from the *old*, still-loaded captured prefix on the transition render), and some producers reuse a fixed `"imported"` session id (so no session-id-keyed reset could ever fire) — no session-keyed latch could be made correct, so it was **reverted to the plain `useMemo`** it replaced. Round 2 also caught a `defaultMayBeStale` warning that read the live `tracePlayhead` against a deliberately-sticky, user-controlled `firingIndex` and so fired spuriously after manual stepping (removed); a sticky-firing key missing `traceSessionId` (a new session with the same node selected silently carried the old session's stepper index); and a truncated-prefix zero-firings case that misleadingly said "did not fire" rather than "not within the loaded prefix." Every fix across both rounds was **mutation-verified** (temporarily reintroduced, confirmed the new test failed, restored). **Lesson for future chunks:** a fresh adversarial pass on the *already-fixed, already-committed* diff is not redundant — it caught that a correctness fix's own optimization had silently regressed correctness, something no amount of re-reading the same fix in place would surface. CI green Ubuntu + Windows × Python 3.12 + 3.13 (4 legs) on both commits.

**Phase 10.6 shipped at main=`99e7bd1` (PR #59).** File-watcher on `grackle serve --watch` (ADR-0027): stdlib mtime-polling by default (`_watch_poll`), optional accelerated `watchfiles` backend behind `try: import watchfiles` (never a required dependency). On a real content change, hash-gates (SHA-256-of-bytes, with a cheap `(mtime_ns, size)` pre-filter to skip re-hashing unchanged files) before evicting the cache and re-parsing — atomic-save (write-temp-rename) and a bare `touch` fire zero re-parses. A dedicated single-worker `ThreadPoolExecutor` runs the rebuild off the event loop; `_watch_loop` skips parse+broadcast entirely when no client is connected. The re-push reuses the **exact same** `protocol.make_static_graph(graph)` builder and `broadcast()` fan-out as the connect-time push — **no new wire message type** (`check-parity` stays a no-op). `--watch-interval` (default 0.3s) is both the poll cadence and the `watchfiles` debounce window; measured warm-cache rebuild ≈55ms on `fixtures/stress-2k` (209 files), comfortably inside the interval. **Deliberately incomplete by design:** the re-push is wire-indistinguishable from a fresh connect, so before 10.7 the frontend rebuilt Sigma from scratch on every watch event — scrambling FA2 layout positions and resetting the camera. ADR-0027 explicitly defers the fix to 10.7 ("10.6's job is only to get a correct, timely re-push to the browser"). CI green Ubuntu + Windows.

**Phase 10.7 shipped at main=`324ca01` (PR #60).** The frontend payoff of 10.6: a watch-mode `static_graph` re-push now diffs against the **live graphology instance** instead of rebuilding Sigma from scratch, so ForceAtlas2 layout positions and the camera survive edits. **Frontend-only, no wire-schema change.** **Mechanism:** a new pure `graph/applyGraphDiff.ts` merges survivor wire attributes (kind/name/path/line/metadata/label) **without ever touching x/y/size/color/hidden**, seeds new nodes near a positioned neighbor (falling back to the graph centroid, then RNG), and multiset-matches parallel edges by a `source\0target\0kind\0line` composite key (edges have no wire id, and 10.4's parallel-edge preservation means `(source,target,kind)` alone isn't unique) — a line-only change is correctly a drop+add, not an in-place update. `GraphCanvas.tsx` splits into an unmount-only teardown effect plus a rebuild-or-apply effect gated by a `hasSurvivor` check (≥1 shared node id ⇒ diff incrementally; zero ⇒ scratch-rebuild, e.g. a genuinely different project); new nodes/edges pulse in and removed nodes fade out via a small rAF-driven `graph/graphAnimation.ts` (easeOutBack overshoot, `#rrggbb`-only color lerp per ADR-0015), all suppressed under `prefers-reduced-motion` (adds appear instantly, removes drop synchronously). A bounded FA2 "reheat" pins survivors (`fixed: true`, which `graphology-layout-forceatlas2` reads directly) so only the changed neighborhood resettles. `useGraphStore.setGraph` now preserves `selectedNodeId` across a re-push when the node survives (still unconditionally clears edge selection/highlights/diff overlay — the diff-baseline persistence contract from 9.3 depends on that clear). **Pre-existing bug fixed en route:** the node reducer returned a bare `{color, size, hidden}`; Sigma's `nodeReducer` *replaces* attrs wholesale rather than merging, so `x`/`y` were silently dropped and Sigma threw on its first `addNode` — **the graph canvas had never actually rendered**, on `main` too, isolated only by PR #46's `ErrorBoundary`. Fixed by spreading `{...data}` before the computed display fields. **Review (two full xhigh multi-agent rounds):** round 1 (7 finder angles → verify → sweep) found and fixed two real bugs in the reheat/FA2 state machine — a node-changing re-push arriving *during* the initial 5s settle cleared the shared stop-timer and pinned survivors mid-settle, freezing a permanently half-settled layout (fixed by splitting into `initialSettleTimerRef`/`reheatTimerRef` and having `reheat` no-op while the initial settle is pending); and `reheat`'s bare `fa2.start()` silently no-ops when FA2 is already running (confirmed in the library source), so survivor pins reached the simulation only via an incidental worker respawn — fixed with `fa2.stop(); fa2.start()` to force a matrix rebuild. A candidate "space-separator collision" in the edge composite key was **refuted** by direct byte inspection (the delimiter is `\0`, which review tooling had rendered as whitespace). Round 2, run specifically to distrust round 1's fixes (per the 10.5 lesson), confirmed them clean. **Verified live** against a real `grackle serve --watch` session: survivor positions bit-for-bit identical across add/edit-line/delete/atomic-no-op-save cycles; zero nodes left pinned when an edit interrupts the initial settle. CI green Ubuntu + Windows × Python 3.12 + 3.13 (4 legs).
