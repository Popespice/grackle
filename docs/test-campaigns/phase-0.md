## Findings

### F1 — Binary frames crash handler (MEDIUM)

**Location**: [packages/agent/src/grackle/server.py:30](packages/agent/src/grackle/server.py:30)

**Reproducer**: send a binary frame with non-UTF-8 bytes (`\x00\x01\x02\xff`)
to the running server.

**Observed**: server logs `connection handler failed` + a full
`UnicodeDecodeError` stack trace, then closes the connection with WebSocket
close code **1011 (internal error)**.

**Expected**: silently drop the frame (the documented pattern for malformed
input) and keep the connection open, or close with the more accurate **1003
(unsupported data)** code.

**Fix**: wrap the decode in `try` and `continue` on failure, or use
`raw.decode("utf-8", errors="replace")` and let `parse_envelope` reject it
through the normal `InvalidEnvelope` path:

```python
async for raw in ws:
    try:
        msg = raw.decode() if isinstance(raw, bytes) else raw
    except UnicodeDecodeError:
        continue
    try:
        envelope = protocol.parse_envelope(msg)
    except protocol.InvalidEnvelope:
        continue
    ...
```

**Why it matters**: 1011 is a misleading close code (suggests *our* server
crashed), and a stack trace in the log will look like an outage to whoever
reads it. The existing `test_abnormal_close_does_not_crash_server` test
catches abrupt TCP disconnects but not invalid UTF-8.

**Recommendation**: ship the fix + add a `test_binary_frame_is_dropped` case.
The existing test file already imports everything needed.

### F2 — `packages/agent/uv.lock` is untracked (MEDIUM)

**Reproducer**: `git ls-files packages/agent/uv.lock` returns empty;
`ls packages/agent/uv.lock` shows the file exists.

**Observed**: the file has been generated by `uv sync` but never committed,
even though `.gitattributes` already declares `uv.lock merge=binary` (a rule
that only applies to *tracked* files).

**Why it matters**: CI runs `uv sync` from scratch on every job. Without a
committed lockfile, dependency resolution is free to pick newer compatible
versions than what passed tests locally. The `--frozen` discipline that
`pnpm install --frozen-lockfile` enforces in CI for the JS side has no Python
equivalent currently.

**Fix**: `git add packages/agent/uv.lock && git commit -m "build(agent):
commit uv.lock for reproducible installs"`. Update the CI step to use
`uv sync --frozen` (or `--locked`).

### F3 — First commit uses scope `repo` outside the enum (LOW)

**Location**: commit `cabb5e8` (chunk 0.A) — `feat(repo): initial scaffold...`

**Why it matters**: `commitlint.config.cjs` lists scopes as `[agent, frontend,
shared-types, docs, ci, tooling, release]` — `repo` is not present. Any
future commit using `feat(repo): …` would be blocked at the hook.

**Fix options**: (a) accept the historical inconsistency (chunk 0.A predates
the lint config, by design); (b) add `repo` to the enum for future use; (c)
amend history (not worth the churn for one commit).

**Recommendation**: do (a). It's documented now and won't recur.

### F4 — Generated `_generated/messages.py` has a duplicate TypedDict import (COSMETIC)

**Location**: [packages/agent/src/grackle/_generated/messages.py:6](packages/agent/src/grackle/_generated/messages.py:6)

```python
from typing import Any, Literal, NotRequired, TypedDict
from typing_extensions import TypedDict   # shadows the first import
```

The file is gitignored, never imported by application code (per
`protocol.py`'s module docstring), and mypy-excluded — so this has zero
runtime effect. It comes from `datamodel-code-generator` and is reported
upstream-style noise. Worth knowing only as background context.

### F5 — `--host 0.0.0.0` binds without a security warning (LOW)

**Reproducer**: `uv run grackle serve --host 0.0.0.0` — binds to all
interfaces silently. `netstat -an | grep 7878` confirms `*:7878 LISTEN`.

**Why it matters**: ADR-0002 says the server is "local-only safe" and binds
to `127.0.0.1` only by default. A user who passes `--host 0.0.0.0` for a
quick test exposes the agent to the network without any prompt.

**Fix**: log `WARNING: binding to non-loopback address; agent will be
reachable from the network` when `host` isn't `127.0.0.1` / `localhost` / `::1`.

## What worked well

Positive evidence worth recording — these are not "no findings" but
*verified* by active probing:

- **Origin allowlist**: rejects spoofed origins (`example.com.evil.com`),
  honors `GRACKLE_ALLOWED_ORIGINS` env, allows no-Origin native clients
  per WebSocket convention, returns close code **1008** for bad origins.
- **Loopback-only by default**: `netstat` confirms only `127.0.0.1.7878
  LISTEN`. Cross-interface connection (`/dev/tcp/$LOCAL_IP/7878`) refused.
- **Concurrency**: 10 clients × 5 pings = 50 successful pongs in 34 ms; 100
  sequential pings round-trip in 43 ms (~2300 msg/s); message ordering
  preserved end-to-end.
- **Codegen determinism**: two consecutive `pnpm codegen` runs produce
  byte-identical outputs.
- **Cross-platform path discipline**: `ruff` PTH rules trigger on
  synthesized `os.path.join` violation; no `os.path.*` or string-path
  `open()` calls in current source.
- **UI reactivity**: live `ConnectionBadge` transitions to red
  `disconnected` (`oklch(0.67 0.22 25)`) within ~100 ms of killing the
  agent; transitions to green `connected` and animates `badge-pulse` when
  agent comes back.
- **Theme system**: toggle persists to `localStorage["grackle:theme"]`,
  survives page reload, respects `prefers-color-scheme` on first visit.
- **Fuzz robustness**: truncated JSON, 1000-level nested payloads,
  injection-style strings (`'; DROP TABLE`, `"><script>`, `🦅emoji`) all
  handled correctly — invalid envelopes dropped, valid ones echoed verbatim.
